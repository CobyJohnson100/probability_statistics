# Coding Standards

## Overview
This document outlines the coding standards and best practices for the probability_statistics Python package. These guidelines ensure consistent, readable, and maintainable code across the project.

## Documentation
- Keep code comments minimal

## Error Handling
- Use appropriate exception types
- Provide clear error messages
- Handle edge cases gracefully

## Testing
- Make high value tests for important functions
- Use descriptive test names

## Version Control
- Commit frequently with clear, concise messages
